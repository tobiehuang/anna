# 塔羅解牌對話機器人 — 上線路線圖

## 現在的進度

已完成的東西有兩個：一份「安娜老師」的角色系統提示詞（機器人的靈魂，之後換任何平台都沿用），以及一個可以直接打開使用的網頁原型《塔羅解牌練習室》。

## 第一階段：自己先試用（今天就能做）

1. 到 platform.claude.com 註冊 Anthropic 帳號，儲值一點金額（例如 5 美元），在 API Keys 頁面建立一把金鑰。
2. 打開《塔羅解牌練習室.html》，貼上金鑰，開始測試。
3. 費用概念：原型使用 Claude Sonnet 5（每百萬 token 輸入 3 美元／輸出 15 美元），一場完整的解牌討論大約幾美分，換算台幣約 1～3 元，5 美元可以練習很多場。
4. 自己扮演學員多測幾輪，記下「老師哪裡教得不像你」，回報後即可調整提示詞。

## 第二階段：修角色（一到兩週）

原型的行為完全由系統提示詞決定。試用時針對這些面向給回饋：引導的節奏會不會太快或太慢、示範的時機對不對、語氣像不像你心目中的老師、牌義的講法有沒有跟你的教學衝突。每一條回饋都能直接改進提示詞。

## 第三階段：給學員用（重要的安全問題）

**目前的原型不能直接把整個檔案（含你的金鑰）發給學員**，因為金鑰存在誰的瀏覽器裡，誰就能用你的額度呼叫 API。給學員用有兩條路：

- 簡單路線：把 HTML 檔發給學員，請每位學員自己申請金鑰、自己付費。零開發成本，但對非技術背景的學員有門檻。
- 正式路線（建議）：架一個小型後端（例如 Vercel 或 Cloudflare 的免費方案），金鑰放在伺服器端，學員打開網址、輸入你發的課程密碼就能用。你統一付 API 費用，也能控管用量。這一步需要寫一點程式，可以再回來請 Claude 做，大約一次對話就能完成。

## 第四階段：加入你的教材

把你的牌義筆記、講義整理成文字檔交給 Claude，會依份量選擇做法：教材在幾萬字以內就直接併入系統提示詞的「牌義系統」段落；份量更大就改用檢索式架構（RAG），機器人回答前先查你的教材。做完之後老師教的就是你的系統，不是通用牌義。

## 之後可以考慮的功能

對話紀錄存檔讓你檢視學員的練習狀況、內建 78 張牌的抽牌功能（學員不用自己洗牌再輸入）、練習成果評分與進度追蹤、換成 LINE 介面觸及更多學員。
